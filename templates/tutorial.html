    hermes-web 基础功能教程

    覆盖 hermes-web Web 控制台全部 26 个模块的操作指南

    账户与登录

    登录与账户

    位置:根路径 /(公开页面)

    两种登录方式

    打开 Web 控制台后,默认进入登录页。系统支持以下两种登录入口,根据你的部署情况选其一即可:

    1. 令牌登录:把管理员提供的访问令牌粘贴到登录框 → 点「登录」。
    2. 用户名 / 密码登录:默认账户信息由管理员在系统初始化时设定(首次登录后系统会强制提示你修改)。连续输错密码会被临时锁定,过几分钟再试或联系管理员。

    登录后立即要做的两件事

    • 修改默认账户:登录后会弹出「请修改默认账户和密码」横幅 → 点「去修改」进入 设置 → 账户,修改用户名与密码。
    • 切换语言:左下角主题切换区域旁的语言下拉,或浏览器右上角地球图标(支持中 / 英 / 日 / 韩 / 法 / 德 / 西 / 葡 / 俄 / 繁中等)。
    忘记密码 / 被锁定:联系超级管理员,让其进入 设置 → 账户管理 重置你的密码,或在 设置 中为你的账户重新启用。
    对话与渠道

    AI 对话

    路由:/hermes/chat(或带 sessionId 的会话详情)

    主界面分区

    区域说明
    左侧会话列表所有本地会话,按来源(Telegram/Discord/Slack 等)分组折叠
    中间消息流Markdown 渲染、代码高亮、工具调用折叠
    底部输入框多行输入、文件拖拽、语音输入、Slash 命令

    关键操作

    • 新建会话:点左下角「+」按钮 → 自动选默认 profile/模型。
    • 切换模型:输入框上方下拉;不同 profile 下能看到的模型不同。
    • 上传文件:拖拽文件到聊天区,或点输入框旁的回形针图标。
    • 下载文件:点击消息中由 Agent 生成的「文件链接」,支持本地 / Docker / SSH / Singularity 后端按路径下载。
    • 工具调用详情:每条 Agent 回复下方的「工具调用」可展开参数和返回结果。
    • 中断运行:回复生成中可点「停止」按钮。
    • 会话搜索:按 Ctrl + K 唤起搜索框,只搜本地 Web UI 会话库。
    • 侧边操作:右键会话列表项 → 重命名 / 删除 / 切换 profile。

    顶部标签页标题会自动切换为该会话的标题,便于多任务切换。

    全局 Agent

    路由:/hermes/global-agent

    与「AI 对话」共用 ChatPanel 组件,但绑定到一个 共享的全局会话上下文:

    • 所有用户/会话在同一全局会话池中可被 Agent 检索和引用。
    • 适合做「跨会话长期记忆 + 检索增强」的场景。
    • 操作方式与会话页一致:新建全局会话、消息收发、文件上传、会话搜索均通用。

    详见 AI 对话 章节的操作清单。

    历史会话

    路由:/hermes/history

    浏览 Hermes Agent 在各平台(Telegram / Discord / Slack / 桌面端等)产生过的全部历史会话(只读浏览,不可编辑)。

    功能

    • 按来源(Telegram / Discord / Slack / CLI / Web 等)分组展示。
    • 点击会话查看完整历史消息(含工具调用)。
    • 导入到本地:将某条 Hermes 历史会话复制到本地 Web UI 会话库,便于继续编辑。
    • 批量删除:勾选多个会话一次性删除。
    • 大纲侧栏:长会话自动生成章节大纲(OutlinePanel)便于跳转。

    群聊

    路由:/hermes/group-chat(或带 roomId 进入指定房间)

    多 Agent 聊天室,支持多个 Agent 在同一房间协作回答。

    操作步骤

    1. 创建房间:点「+ 新建房间」→ 输入房间名 → 创建。
    2. 添加 Agent:进入房间 → 右上角「成员管理」→ 选择已有 Agent 并绑定其 profile。
    3. 邀请码:房间设置中可生成邀请码分享给他人加入。
    4. @提及:在消息中使用 @agent-name 触发指定 Agent 回复;未 @ 时所有 Agent 可见但不主动回复。
    5. 上下文压缩:当历史超过 token 阈值时自动摘要(可在「设置 → 压缩」调整阈值)。
    6. 实时反馈:输入 / 打字 / 回复进度指示器实时显示。
    7. 删除房间:房主可在设置中解散。

    平台渠道

    路由:/hermes/channels

    统一管理 8 个外部平台 的接入凭证与行为设置。所有改动在页面点「保存」后立即生效。

    平台关键配置备注
    TelegramBot Token、@提及控制、Reaction、自由回复需先在 Telegram 创建 Bot 并取得 Token
    DiscordBot Token、@提及、自动开 Thread、Reaction、频道白/黑名单
    SlackBot Token、@提及、机器人消息处理
    WhatsApp启用开关、@提及控制、提及正则
    MatrixAccess Token、Home Server、自动 Thread、私信 @ 线程
    飞书 (Lark)App ID / Secret、@提及控制
    微信浏览器扫码登录 → 自动保存凭证仅桌面端可用
    企业微信Bot ID / Secret

    状态指示

    每个平台卡片顶部会显示「已配置 / 未配置」,未配置时只需填写对应字段 → 点保存即可激活。

    模型与配置

    模型管理

    路由:/hermes/models

    两个子面板

    1. Providers(提供商)
    • 列出当前 profile 可见的所有模型提供商(预设 + 自定义 OpenAI 兼容)。
    • 操作:新增 / 编辑 / 删除提供商;自定义 URL(自动识别 /v1、/v4 等非 v1 版本)。
    • OAuth 一键登录:OpenAI Codex、Nous Portal、GitHub Copilot。
    2. Auxiliary Models(辅助模型)
    • 默认/备用模型选择;分组管理同一提供商的多个模型。
    • 点选切换该 provider 下的默认模型。

    配置(Profile)

    路由:/hermes/profiles · 权限:仅超级管理员

    Hermes 的「工作环境」概念。每个 profile 是独立的配置 + 缓存 + 上传 + 会话 + 任务等容器。

    操作步骤
    新建右上角「创建」→ 输入名称 → 创建
    重命名行内菜单「重命名」
    克隆行内菜单「克隆」→ 输入新名字(继承所有配置与缓存)
    删除行内菜单「删除」→ 二次确认
    导入右上角「导入」→ 选择压缩包文件(来自其他部署或备份)
    导出行内菜单「导出」→ 下载压缩包文件用于迁移/备份
    切换点击左侧 profile 名(顶部 sidebar 的头像区也可切换)
    切换 profile 后,会话、模型、上传、看板、任务全部切换到对应 profile。普通管理员只能看到与自己账户绑定的 profile;超级管理员可见全部。
    工具与协作

    文件浏览

    路由:/hermes/files

    支持多种后端的文件管理:本地 / Docker / SSH / Singularity。

    操作清单

    • 浏览树 + 平铺切换;面包屑跳转。
    • 新建文件 / 新建文件夹:右键空白区域或工具栏按钮。
    • 重命名 / 复制 / 移动 / 删除:右键文件行。
    • 上传:工具栏「上传」→ 选择多个文件,落到当前 profile 的 upload 目录。
    • 下载:直接点文件名(路径解析式下载,可下载上传目录之外的 Agent 生成产物)。
    • 预览:内置代码高亮、图片预览、文本查看器;右键可「编辑器」中修改保存。

    定时任务

    路由:/hermes/jobs

    Cron 表达式驱动的自动化任务(绑定到当前 profile)。

    操作流程

    1. 右上角「新建任务」→ 填写:名称、描述、Cron 表达式(提供「每小时 / 每天 / 每周」等快捷预设)、触发时要发送给 Agent 的提示词。
    2. 列表视图:每个任务卡片展示状态、上下次执行时间。
    3. 立即执行:点「▶ 运行」可手动触发,不等 Cron。
    4. 暂停 / 恢复:行内切换按钮。
    5. 编辑 / 删除:行内操作。
    6. 运行历史:下方面板展示每次执行的开始时间、耗时、结果摘要。
    7. 排序:顶部可按名称 / 时间排序。

    看板

    路由:/hermes/kanban

    Agent 任务规划与跟踪看板(profile 隔离)。

    默认九个状态列

    triage → todo → scheduled → ready → running → blocked → review → done → archived

    操作

    • 新建任务:右上角「+ 新建」→ 填标题、描述、指派人、初始状态。
    • 拖拽:在状态列之间拖动卡片改变状态。
    • 点开卡片:右侧抽屉显示详情 + 评论 + 子任务。
    • 创建看板:默认看板不可删,可创建自己的副板(顶部「看板」下拉切换)。
    • 多看板切换:URL ?board=xxx 或顶部下拉。
    • 自动刷新:默认定时刷新任务状态(可关)。

    工作流

    路由:/hermes/workflow

    可视化多 Agent 协作流程图编辑器(基于 Vue Flow)。

    核心能力

    • 节点类型:起始、Agent、工具、条件、结束等。
    • 连线:拖拽节点边框到目标节点建连接,附带条件表达式。
    • 侧边聊天:右侧抽屉集成聊天面板,可在编辑时直接对 Agent 调试节点。
    • 保存 / 另存为新版本:每次保存生成版本号。
    • 立即运行:点工具栏「▶ 运行」触发;实时查看节点状态、Token 用量。
    • 运行记录:下方面板可看历史 run,从指定节点重新运行(rerun-from-node)。
    • 停止运行:运行中可手动终止。
    • 批量操作:多选后批量删除工作流。
    Agent 能力

    专家中心

    路由:/hermes/experts、/hermes/experts/:slug

    浏览 / 安装 / 使用官方或社区专家(Expert)和团队(Team)配置。

    三个 Tab

    1. Published(已发布):线上专家市场,可按分类筛选、搜索。
    2. Team(团队):已购买的团队方案。
    3. Installed(已安装):本地已安装的专家/团队。

    单个专家详情页

    • 顶部 Hero 区:封面、描述、技能列表、推荐启动提示词。
    • 点「对话」按钮 → 直接以该专家人设开启新会话。
    • 点「安装 / 卸载」管理本地安装状态。

    编码代理

    路由:/hermes/coding-agents

    在 Web 中启动并监控本地 Claude Code / Codex 编码 Agent。

    配置流程

    1. 顶部选择 Claude Code 或 Codex。
    2. API 模式:自动 / Anthropic 兼容 / OpenAI 兼容 三选一。
    3. 配置文件:可视化编辑该工具的 config 文件(保存立即生效)。
    4. 启动:选择本地终端或内置终端面板启动。
    5. 运行中:实时显示 stdout/stderr、推理过程(reasoning);超时与重启策略可调。
    6. 会话保存:每次会话的输出与推理元数据会自动保留,可在历史列表中随时回看。

    技能

    路由:/hermes/skills

    浏览当前 profile 下安装的所有 Hermes Skills。

    功能

    • 分类侧栏:按目录展示;可搜索。
    • 目标切换:顶部「Hermes / Claude / Codex」切换不同 Agent 看到的技能集合。
    • 详情抽屉:点技能 → 看说明、关联文件、依赖。
    • 导入:右上角「导入」→ 上传技能压缩包文件。
    • 外部目录:可添加外部技能目录路径。
    • Pending Write Approvals:底部徽标显示等待写权限审批的数量(来自 Write Gate),点开抽屉可批量批准或拒绝。
    • 已归档:底部 .archive 分类查看历史版本。

    技能使用记录

    路由:/hermes/skills-usage

    统计每个技能在最近会话中 被调用次数、成功率、平均耗时 等指标,用于评估技能实用度。

    插件

    路由:/hermes/plugins

    查看当前 profile 已安装的 Hermes 插件。

    列表字段

    • 来源(bundled / marketplace / 自定义)
    • 类型(kind)
    • 有效状态:enabled / auto-active / inactive / disabled / provider-managed
    • 搜索 + 多维筛选。

    顶部摘要卡片显示:总数、激活数、未激活数、禁用数、Provider 管理数。

    记忆

    路由:/hermes/memory

    维护 Agent 的长期记忆(三段 Markdown):

    段落用途
    memory通用长期事实
    user用户偏好与画像
    soulAgent 人设与价值观

    操作

    • 只读浏览:默认折叠为卡片预览。
    • 点「编辑」进入 Markdown 编辑器 → 保存立即生效。
    • 切换 profile 后看到的是该 profile 独立的记忆文件。
    数据与统计

    使用统计

    路由:/hermes/usage

    查看当前 profile 的 Token 用量与会话成本。

    指标

    • 总览卡片:总 Token(输入 / 输出)、会话总数、日均会话、预估成本、缓存命中率。
    • 模型分布饼图:每个模型占用比例。
    • 每日趋势:支持 7d / 30d / 90d / 365d 切换;柱状图 + 下方明细表。

    性能监控

    路由:/hermes/performance · 权限:仅超级管理员

    实时观察 Web UI 后端 + Hermes Agent Bridge 的运行时状态。

    显示项

    • Web UI 进程 RSS 内存、CPU。
    • Bridge Broker 进程内存、活跃 Worker 数。
    • Socket.IO 连接数。
    • 自动刷新开关。

    日志

    路由:/hermes/logs

    统一查看 Web 控制台与 Hermes Agent 在运行过程中产生的所有日志条目。

    操作

    • 日志文件下拉:选择 agent / server / error 等文件,显示大小与最后修改时间。
    • 行数:50 / 100 / 200 / 500。
    • 级别过滤:ERROR / WARNING / INFO / DEBUG。
    • 关键字搜索:对消息内容做大小写不敏感过滤。
    • HTTP 访问日志高亮:结构化识别并加色。
    设置与高级

    设置

    路由:/hermes/settings(多 Tab,通过 URL ?tab=xxx 切换)

    Tab说明
    account当前账户改名 / 改密码 / 移除密码登录
    users仅超级管理员:用户管理(增删、改角色、绑 profile、启用/禁用)
    display流式输出、紧凑模式、是否展示推理、是否展示成本
    agent最大回合数、超时、强制工具调用策略
    gateway网关自启动策略
    memory记忆开关、字符上限
    compression长上下文自动压缩阈值、压缩模型
    session会话空闲超时、定时重置
    privacyPII 脱敏规则
    models默认模型 + 默认提供商
    voiceTTS / STT 提供商、声音、克隆参考音频

    Web 终端

    路由:/hermes/terminal · 权限:仅超级管理员

    基于 xterm.js + node-pty 的浏览器内终端。

    操作

    • 多会话:可同时开多个终端面板,顶部下拉切换。
    • 主题切换:Default / Solarized Dark 等内置主题。
    • 自适应大小:浏览器窗口缩放自动 resize PTY。
    • 关闭会话:点 X 关闭,确认后会结束对应的终端面板连接并释放资源。

    设备发现

    路由:/hermes/devices

    局域网内 Hermes 设备的发现、配对、远程命令。

    功能

    • 扫描:点「扫描」按钮搜索局域网内可发现的 Hermes 设备。
    • 配对:选中设备 → 「配对」→ 输入验证码 / 扫码 → 设备出现在列表中。
    • 手动配对:粘贴对端提供的 Pairing URL 完成远端加入。
    • 审批 / 拒绝 / 屏蔽:列表行内操作。
    • 历史请求:抽屉显示收到的远端请求与已执行命令。
    • 复制配对链接:便于发给其他机器。

    MCP 管理

    路由:/hermes/mcp · 权限:仅超级管理员

    管理本地 MCP(Model Context Protocol)服务器,供所有 Agent 调用工具。

    操作

    1. 新增服务器:右上角「+」→ 选择 JSON / YAML 输入模式 → 填入 command / args / env 等。
    2. 测试连接:保存前点「测试」验证服务可启动。
    3. 工具可见性:点「工具」按钮 → 选择 all / include / exclude + 具体工具名,控制暴露给 Agent 的工具集。
    4. 热重载:点「Reload」让所有 Agent 重新读取 MCP 工具清单。
    5. 删除:行内「删除」按钮。

    版本预览

    路由:/hermes/version-preview · 权限:仅超级管理员

    在不破坏当前主版本的前提下,试用 GitHub 上某个 commit / PR 的预编译版本。

    操作

    • 填入 GitHub 引用(branch / tag / commit SHA / PR 号)。
    • 点「拉取并启动预览」→ 后台下载并并行启动预览实例,不影响主服务。
    • 预览结束后可一键回滚。
    适合 QA、回归测试、给客户演示未发布特性。

    配网步骤

    1

    连接设备热点

    手机或电脑连接 QH 开头的 WiFi

    设备接通电源后自动进入AP 模式,请按以下步骤操作:

    1. 打开手机或电脑的 WiFi 设置,搜索可用网络。
    2. 找到名为 'QH' 开头 的 WiFi 热点,点击连接。
    3. 连接成功后,打开浏览器,在地址栏输入 192.168.4.1 进入配网页面。
    如果搜不到热点,请确认设备已接通电源。热点在设备开机后约 30 秒内出现。
    2

    填写配网信息

    输入 Wi-Fi 名称、密码及手机号

    进入配网界面后,请按照图中标注填写:

    1. 在 Wi-Fi 名称栏中选择或输入您家中/公司的无线网络名称。
    2. 在 密码栏中输入对应的 Wi-Fi 密码。
    3. 在 电话号码栏中输入您的手机号(用于设备绑定)。
    4. 确认信息无误后,点击"保存并连接"按钮。
    请确保 Wi-Fi 密码输入正确,否则设备将无法连接网络。等待 5 至 10 秒即可完成。
    3

    配网成功

    设备网络切换提醒

    点击"保存并连接"后等待 5 至 10 秒,会出现以下变化:

    1. 页面显示设备网络切换提醒,表示设备正在切换到您配置的网络。
    2. 您的手机或电脑的 WiFi 会自动切回到家中/公司的网络。
    3. 以上现象即代表配网成功。
    配网成功后,设备会自动与您的账户绑定。后续所有管理操作均通过云端后台完成,无需重复配网。
    4

    进入管理平台

    访问 keli.quantclaw.vip

    配网成功后,在电脑或手机浏览器中进行以下操作:

    1. 在浏览器地址栏输入 https://keli.quantclaw.vip 进入设备管理平台。
    2. 在管理平台右上角选择要使用的设备。
    3. 选择好设备后,点击右下角的"控制面板"按钮。
    请确保电脑/手机与设备处于同一网络环境下,否则无法正常访问管理平台。
    5

    Hermes 使用界面

    进入智能管理界面

    点击"控制面板"后,即可进入 Hermes 使用界面:

    1. 界面展示了设备的管理与对话功能区。
    2. 在此可以管理设备模型、查看对话记录等。
    3. 如需为设备配置 AI 模型,请参考基础功能教程 → 模型管理。
    Hermes 是量迹座舱的核心管理界面,后续所有模型配置与对话均在此完成。
